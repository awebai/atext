# Profile authority fidelity

When normalizing a fused agent soul into a reusable role profile, model three
things separately:

1. **Decision rights**: what the role is actually authorized to decide and
   record.
2. **Required evidence**: which peers or systems must supply code, live-state,
   customer, market, or operational facts before the decision is sound.
3. **Execution rights**: which role implements, releases, publishes, or performs
   irreversible work after the decision.

Do not convert mandatory peer evidence into a consensus requirement that erases
the source role's authority. Likewise, do not let decision authority absorb a
peer's implementation or production gate. Preserve disagreement and escalation
rules explicitly, then test the materialized instructions for those boundaries.

For senior roles, add templates for the artifacts that make authority durable,
such as decision records and evidence-bounded release claims. A mission summary
alone is too easy to flatten into generic coordination behavior.
